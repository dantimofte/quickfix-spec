Detects shorthand <component> tags (e.g., <component name="compName1" />) and suggests replacing them with their full definition from the <components> section of the same XML file.

Example


<message>
  <field name="field1"/>
  <component name="compName1" />
  <field name="field2" />
</message>

<components>
  <component name="compName1">
    <field name="aa" />
    <field name="bb" />
  </component>
</components>

Suggestion: Replace component reference with definition

Result:


<message>
   <field name="field1"/>
    <component name="compName1">
      <field name="aa" />
      <field name="bb" />
    </component>
   <field name="field2" />
</message>
